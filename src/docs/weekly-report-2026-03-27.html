FaaS 2.0 — Forms as a Service
6 new insights
Critical Gap Prasanjit · Mar 25

No form preview before student deployment. CRNA programs use 4-level hierarchical controls, repeater fields, and matrix layouts with embedded time pickers. A configuration error in any of these is invisible until students submit. Form simulator is the single highest-value missing FaaS feature.

"We used to have color coding. Now everything is black and white. Students get lost in long forms." — Prasanjit, Mar 25
So WhatPreview/simulator is not a nice-to-have. It is the difference between 1 reviewer catching a mistake and 200 students experiencing it.
Critical Gap Prasanjit · Mar 25

FaaS validation timing is broken: mandatory field errors appear only on submit, not during input. Numeric limits (e.g., minutes over 100) validated only at submission. ICD/CPT selections display as long text strings, not chips. Delete buttons unclear. Real-time field validation is table stakes in 2026.

So WhatEvery field error discovered at submit is a submission abandonment risk. Especially critical for CRNA and nursing programs using long complex forms.
Decision Akshit Q2 · Mar 25

FaaS template-first confirmed by usage data: 80-85% of changes are incremental (add questions/options to existing templates), 2-3% from scratch, 90% start from predefined templates. Current three-card entry point (AI import / library / scratch) must be reordered. Template library = primary. AI import = secondary. Scratch = tertiary.

So WhatRedesign the creation entry screen. Template library leads. Do not bury the most common workflow behind the newest feature.
Blocker Akshit Q2 · Mar 25

No consolidated requirements document exists for FaaS Q2. Phase 1 targets internal users only (CI team, CIC exec, page server team). Workflows hard-coded by tech team. Ankit provides Q2 dependencies end of March. Design update is blocked until that document lands.

So WhatDesign cannot start Phase 1 work without the requirements doc. This is a Q2 schedule risk.
Theme Prasanjit · Mar 25

Domain experience tiers confirmed: PA — simple controls, clone-based. Nursing/CRNA — 4-level hierarchical, repeaters, advanced validation. SLP — matrix-heavy with time calculators. Same FaaS engine, three radically different domain experiences. One-size UI is the core problem.

So WhatSelf-service must show the right complexity tier per domain context. Progressive disclosure: PA sees simple builder, CRNA sees advanced.
Post-Course Evaluation (PCE) — New Workstream
5 new insights
AI Differentiator Strategic Monil · Mar 26

PCE AI differentiation stack: sentiment analysis of free-text, automatic improvement vs compliment categorization, comment filtering with summary generation, SWOT visualization from responses, faculty vs program average comparisons, cross-program trend identification. Competitors Watermark and Explorance Blue deliver only median and mode. No explicit "AI-powered" branding.

So WhatSWOT generation from survey data is first-in-class. Design the analytics layer first — it is the demo hook and the competitive moat.
Architecture Monil · Mar 26

PCE distribution workflow: select template → choose course offering → system auto-populates students and faculty from Prism → guest faculty managed through course offering updates → supplemental questions (1-3 custom) allowed → section toggle (course eval vs faculty eval independently). End of April full design target. Engineering handoff May 2026.

So WhatAuto-population from Prism is the feature that separates PCE from SurveyMonkey. This must be the design centerpiece of the distribution flow.
Market Mohil · Mar 24

Programs currently pay separate vendors (CoreEval, Anthology, SmartEval) approximately $5K/year for post-course evaluation. PCE is priced as a separate paid add-on. Non-subscribers do not see the PCE tile in the UI — monetization gate built into navigation.

So WhatPCE displaces a $5K/yr spend per program. Tile visibility is the product monetization signal — not just a UX detail.
Open Question Monil · Mar 26

Three open questions blocking distribution workflow design: (1) who distributes surveys — PD, course coordinator, or admin? (2) student response rate tactics — incentives or gamification? (3) cross-faculty comparison fairness methodology. Competitor walkthroughs (Watermark, Explorance Blue) scheduled next week.

So WhatTemplate and analytics layers can proceed. Distribution workflow design is blocked pending persona decision on Question 1.

Platform Signal NPS 2025 · 1,282 responses

Top 5 student detractor themes from textual responses: (1) navigation — cannot find hour tracking, features in unexpected locations; (2) mobile — clocking in/out requires too many steps; (3) preceptor form length discouraging completion; (4) sign-in friction; (5) compliance false positives showing requirements as missing after program cleared them.

So WhatNavigation and mobile are the top 2 design levers for NPS recovery. Nursing accounts for 47% of volume — Nursing-first fixes move the overall metric most.
Insight NPS 2025 Approve · 8 responses

Approve reached approximately 9.6 average NPS — the highest-satisfaction product in the portfolio. Top valued attributes: expertise on requirements guidelines, responsiveness, turnaround time. Managed service model succeeds where self-service fails. The lesson: human expertise + speed = promoter.

So WhatEvery FaaS self-service feature should ask: can this match what Approve does manually? If not, it will generate tickets, not promoters.
